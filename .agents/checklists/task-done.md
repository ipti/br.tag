# Checklist: Task Done

Use this before closing implementation work.

- The change matches existing project architecture.
- New feature code was placed in a module when applicable.
- Commands and examples used repo-verified scripts where possible.
- User-visible claims are backed by code references or test evidence.
- Relevant verification was run or explicitly reported as skipped.
- If SASS changed, `composer run sass:build` was considered.
- If behavior changed, correlated flows were at least reviewed.
- If routing or module structure changed, menus/layouts and legacy `?r=` references were considered.
